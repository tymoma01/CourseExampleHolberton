CSS selects HTML elements and applies styles. It controls color, layout, spacing, and everything you see on the screen.

/* Selecting an element */
h1 {
  color: #2a5f8f;
  font-size: 2rem;
}

/* Selecting by class */
.card {
  background: white;
  padding: 1.5rem;
  border-radius: 8px;
}

/* Pseudo-class on hover */
a:hover {
  text-decoration: underline;
}

Selector

A selector targets the HTML element(s) you want to style. It can be a tag name, a class, an ID, an attribute, or a combination of these.

💡 Use classes (.my-class) over IDs (#my-id) for maximum reusability — IDs must be unique per page.

The Properties

Core CSS Concepts

element
Type Selector
Targets all instances of an HTML tag. e.g. p { } styles every paragraph on the page.
Specificity: 0-0-1
.class
Class Selector
Targets all elements with a given class attribute. Reusable — the preferred way to style in modern CSS.
Specificity: 0-1-0
#id
ID Selector
Targets a single unique element. Very high specificity — use sparingly and prefer classes instead.
Specificity: 1-0-0
*
Universal Selector
Targets every element on the page. Often used in resets: * { box-sizing: border-box; }
Specificity: 0-0-0
:hover
Pseudo-class
Targets an element in a specific state. Common: :hover, :focus, :visited, :nth-child(). Remember LVHA order for links.
Specificity: 0-1-0
::before
Pseudo-element
Targets a specific part of an element. Uses double colon (::). Common: ::before, ::after, ::first-letter.
Specificity: 0-0-1
A > B
Child Combinator
Targets B only when it is a direct child of A. More specific than the descendant combinator (space).
Combinator
[attr]
Attribute Selector
Targets elements with a given attribute. e.g. a[href^="#"] targets all internal anchor links.
Specificity: 0-1-0
/* Type, class, and ID selectors */
p         { color: black; }     /* 0-0-1 */
.card     { padding: 1rem; }  /* 0-1-0 */
#hero     { height: 100vh; } /* 1-0-0 */

/* Combinators */
nav a     { color: white; }    /* descendant */
ul > li   { list-style: none; } /* direct child */
h2 + p   { margin-top: 0; }   /* adjacent sibling */

/* Pseudo-classes — LVHA order for links */
a:link    { color: blue; }
a:visited { color: purple; }
a:hover   { text-decoration: underline; }
a:active  { color: red; }

THE BOX MODEL

MARGIN
BORDER
PADDING
CONTENT
margin
Margin
Space outside the border. Pushes other elements away. Margins can collapse between adjacent elements.
Box Model · Outside
padding
Padding
Space between the content and the border. Always takes the background color of the element.
Box Model · Inside
border
Border
The line around an element. Shorthand: border: 2px solid red. Use border-radius to round corners.
Box Model · Edge
box-sizing
Box Sizing
Controls how width/height are calculated. Use border-box so padding and border are included in the declared width.
Box Model · Critical
width / height
Dimensions
Sets the size of an element's content area. Use max-width, min-width for responsive layouts.
Box Model · Size
display
Display
Controls how an element is rendered. Key values: block, inline, inline-block, flex, grid, none.
Layout · Critical
/* Best practice reset */
*, *::before, *::after {
  box-sizing: border-box;
}

.card {
  width: 300px;
  padding: 1.5rem;       /* inside space */
  margin: 0 auto;       /* centered horizontally */
  border: 1px solid #ccc;
  border-radius: 8px;
}

/* With border-box: total width = 300px */
/* Without it:      total width = 300 + padding + border */
color
Color
Sets text color. Accepts named colors, hex (#ff0000), rgb(), hsl(), or CSS variables.
Typography
font-size
Font Size
Sets text size. Prefer rem (relative to root) over px for accessibility. 1rem = 16px by default.
Typography · Size
font-weight
Font Weight
Controls text boldness. Values: 100–900 or keywords (normal = 400, bold = 700).
Typography
line-height
Line Height
Space between lines. Use unitless values (e.g. 1.5). Minimum 1.5 recommended for body text accessibility.
Typography · Accessibility
text-align
Text Align
Horizontal alignment of text within a block element. Values: left, center, right, justify.
Typography · Layout
letter-spacing
Letter Spacing
Space between characters. Useful for headings, monospace labels, and ALL CAPS text.
Typography
text-transform
Text Transform
Changes text casing without modifying the HTML. Values: uppercase, lowercase, capitalize.
Typography
--var
CSS Variables
Custom properties defined with -- prefix. Declared on :root for global scope, used with var(--name).
Variables
/* CSS variables on :root */
:root {
  --color-primary: #2a5f8f;
  --font-size-base: 1rem;
}

body {
  font-size: var(--font-size-base);
  line-height: 1.6;       /* unitless — scales with font */
  color: #1a1410;
}

h1 {
  font-size: 2.5rem;
  font-weight: 700;
  letter-spacing: -0.02em;
  text-transform: uppercase;
  color: var(--color-primary);
}
background
Background
Sets element background. Accepts colors, images, or gradients: linear-gradient(to right, red, blue).
Visual
position
Position
Controls placement. Values: static (default), relative, absolute, fixed, sticky. Use with top/right/bottom/left.
Layout · Position
transform
Transform
Visually moves, scales, rotates, or skews an element. Does not affect document flow. GPU-accelerated.
Visual · Animation
animation
Animation
Shorthand for CSS keyframe animations. Requires @keyframes rule. animation-name and animation-duration are required.
Visual · Animation
opacity
Opacity
Sets element transparency from 0 (invisible) to 1 (fully visible). Affects the element and all its children.
Visual
box-shadow
Box Shadow
Adds shadow effects around an element. Syntax: offset-x offset-y blur spread color. Use multiple shadows separated by commas.
Visual · Effect
/* Positioning */
.parent { position: relative; }
.child  { position: absolute; top: 0; right: 0; }

/* Transforms */
.card:hover {
  transform: translateY(-4px) scale(1.02);
}

/* CSS animation with @keyframes */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.hero {
  animation: fadeIn 0.6s ease forwards;
}
