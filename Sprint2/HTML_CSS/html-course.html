↑ Click any tag to learn more

<html>

The root element. Every HTML page is wrapped inside this tag. It tells the browser that the document is HTML and contains all other elements.

💡 Always include the lang attribute for accessibility — e.g., <html lang="en">

The Elements

Core HTML Tags

<header>
Header
The top of a page or section. Usually contains logo, navigation, and search. Consistent across all pages.
Block · Semantic
<nav>
Navigation
A structural element with navigation links. Used for menus, breadcrumbs, and page navigation.
Block · Semantic
<main>
Main
The primary content of the page. Should appear only once per page and never inside header, footer, or nav.
Block · Semantic
<section>
Section
Groups related content together. Usually contains its own heading. Think of it as a chapter of your page.
Block · Semantic
<article>
Article
Self-contained content that could stand alone: blog posts, news articles, product cards, forum posts.
Block · Semantic
<aside>
Aside
Content related to but separate from the main content. Sidebars, pull quotes, related links.
Block · Semantic
<footer>
Footer
The bottom of a page or section. Contains copyright info, secondary navigation, social links.
Block · Semantic
<div>
Division
A generic block-level container. Use it only when no semantic element fits. No inherent meaning.
Block · Generic
<span>
Span
A generic inline container. Used to style a portion of text without adding semantic meaning. Use sparingly.
Inline · Generic
<body>
  <header>
    <nav><a href="/">Home</a></nav>
  </header>

  <main>
    <section>
      <article>
        <h2>My Post Title</h2>
        <p>Content goes here.</p>
      </article>
    </section>
    <aside>Related content</aside>
  </main>

  <footer>© 2025 My Website</footer>
</body>
<h1>–<h6>
Headings
Six levels of headings. Always use in order — never skip from h1 to h4. One h1 per page maximum.
Block · Heading
<p>
Paragraph
A block of text that forms a paragraph. Only use for actual paragraph content, not as a generic container.
Block · Text
<strong>
Strong
Indicates strong importance. Semantically meaningful — not just for making text bold visually.
Inline · Text
<em>
Emphasis
Indicates stress emphasis. Changes meaning: "I love HTML" vs "I love HTML".
Inline · Text
<ul> / <ol>
Lists
Unordered (bullets) or ordered (numbered). Always use <li> inside. Can be nested.
Block · List
<blockquote>
Blockquote
Multi-line quotes from external sources. Use the cite attribute to reference the source URL.
Block · Quote
<code>
Code
Inline fragment of computer code. For blocks of code, wrap <code> inside a <pre> element.
Inline · Text
<mark>
Mark
Highlighted or marked text. Used to indicate relevance or search results on a page.
Inline · Text
<h1>Page Title</h1>
<h2>Section Title</h2>

<p>This is a paragraph with <strong>important text</strong>
and <em>emphasized words</em>.</p>

<ul>
  <li>First item</li>
  <li>Second item</li>
</ul>

<blockquote cite="https://example.com">
  A famous quote from somewhere.
</blockquote>
<a>
Anchor / Link
Creates hyperlinks. Use href for destination. Add rel="noopener" when target="_blank" for security.
Inline · Link
<img>
Image
Embeds an image. Self-closing. Always include alt text and specify width/height to prevent layout shift.
Inline · Self-closing
<picture>
Picture
Wraps multiple <source> elements to serve different image formats or sizes for different devices.
Block · Media
<video>
Video
Native video player. Avoid autoplay. Always provide fallback text and multiple source formats.
Block · Media
<audio>
Audio
Embeds audio content. Supports MP3, OGG, WAV. Use controls attribute to show the player.
Inline · Media
<iframe>
Inline Frame
Embeds external content like maps or videos. Always add a title attribute for accessibility. Use sparingly.
Block · Embed
<!-- Link with security best practice -->
<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">Open link</a>

<!-- Image with alt text and dimensions -->
<img src="photo.jpg"
     alt="A mountain landscape at sunset"
     width="800" height="600">

<!-- Responsive picture element -->
<picture>
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Fallback">
</picture>
<table>
Table
For tabular data only. Use thead, tbody, tfoot, tr, th, td. Always add a caption for accessibility.
Block · Table
<details>
Details
Native accordion element. Paired with <summary>, it shows/hides content without any JavaScript needed.
Block · Interactive

▶ What is the difference between <section> and <div>?

<section> is a semantic element that groups related content with a heading. It tells browsers and screen readers that this is a meaningful block. <div> has no semantic meaning at all — it's just a styling wrapper. Always prefer semantic elements when possible.

▶ When should I use <strong> vs <b>?

Use <strong> when the text is semantically important — a warning, a key term. Use <b> to draw attention without implying importance (like keywords in a summary). Both look bold visually, but their meaning differs for screen readers and search engines.
